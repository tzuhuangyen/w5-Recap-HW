  3. What is the difference between synchronous and asynchronous code in JavaScript?

    Q3. Synchronous code in JavaScript does things one after the other, step by step. It waits for each task to finish before moving on to the next one. On the flip side, asynchronous code lets tasks happen independently without waiting for each other to finish. It uses callbacks, promises, or async/await to manage these tasks. This helps the program to keep doing other things while waiting for certain tasks to complete. It's handy for making programs more efficient and responsive, particularly when dealing with tasks that take a bit of time, like fetching data or handling user input.

  4. What is the difference between a Promise and a Callback in JavaScript?

    In JavaScript, both Promises and Callbacks are used for handling asynchronous tasks, but they have differences in how they work and how you use them. Callback: A callback is a function that you pass as an argument to another function. It runs after a specific task finishes or at a set time. Using callbacks for multiple nested asynchronous operations can make the code hard to read and maintain, leading to what's called "callback hell". Promise: A Promise is an object representing the eventual completion or failure of an asynchronous operation and its result. It has a clearer and more organized syntax, making it easier to read and write asynchronous code. Promises come with methods like .then() and .catch() to handle success and failure, simplifying the chaining of asynchronous tasks. In summary, Promises provide a more structured and readable way to handle asynchronous tasks compared to callbacks. They help avoid callback hell and make asynchronous code more understandable.
